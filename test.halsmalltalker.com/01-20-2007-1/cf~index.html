Okay, now let's consider Attempt #2 below.

Attempt #2 tries to make the content (with more content than the prior attempt) found in the lime box to float to the bottom of Column #1 by using the following styles:

.ColumnBottom { margin-top: 100%; border: 1px solid lime; }

IE 6.x and 7.x renders this attempt correctly however FireFox does not without some slight changes.

FireFox requires the following changes to the styles that worked for IE:

.ColumnBottom { margin-top: 89%; border: 1px solid lime; }

The problem of needing to adjust the Styles based on one browser or the other is a common problem for web development.

For this scenario we chose to use some JavaScript to dynamically change the Styles based on the browser type at runtime. Consider the following code:

if (!browser_is_ie) { setSpecificStyles('.ColumnBottom', 'marginTop', '89%'); }

The JavaScript code above states if the browser is not IE then make the margin-top for the ColumnBottom class 89% rather than the default 100%.
